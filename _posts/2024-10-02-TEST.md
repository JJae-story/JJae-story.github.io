# 초기화란?

# Java에서 초기화(initialization)란 ?

-. 선언한 객체에 최초로 값을 넣어주는 것

만약 초기화를 하지 않는다면 어떻게 될까? 

```java
// int형 변수를 선언하였다.
// 변수를 초기화 하지 않고 출력했을 시에 무슨 에러가 뜨는지 알아보자.
public class initializationPractice {
	public static void main(String[] args) {
		int number; 
		System.out.println(number);
	}
}

// The local variable a may not have been initialized
// 라는 에러가 발생한다.
```

초기화 하는 방법을 알아보자

```java
// 생성자(Constructor)에서 변수 값을 주는 것도 가능하다.

위 에러가 발생한 코드를 참고해보면, int형 변수 number에
초기화를 진행해줘야 정상적으로 출력을 할 수 있다.

int number;
number = 1;

int number = 1; // 선언과 동시에 초기화를 해줄 수 있다.
number = 2;
// 선언된 변수의 값을 바꿀 수 있고, 이를 할당(allocation)이라 한다.
```

이렇게 초기화를 해주면 에러가 발생하지 않는 것을 확인 할 수 있다.

그렇다면 초기화를 하는 이유는 무엇일까?

```java
public class initializationPractice {
	int age;
	String name;
	
	public static void main(String[] args) {
		initializationPractice i = new initializationPractice(); // 인스턴스화
		System.out.println(i.age);
		System.out.println(i.name);
	}
}
// 클래스 영역에 선언한 멤버 변수 age, name은 선언만 하고 초기화가 필요없는 것을 볼 수 있다.
// 클래스 영역에 선언한 변수들은 실행 시 초기화를 하지 않아도 int 타입에 String을 넣는 것을 방지하기 위해
// 컴파일러에서 자동으로 int 형이면 0, String, boolean 형이면 null 값을 할당한다.
// 메소드 안에서 선언된 변수는 자동으로 값이 할당되지 않기 때문에 에러가 뜬 것이다.
```

이렇듯 클래스 영역에서 선언할 경우 컴파일러가 자동으로 값을 할당하여 주지만, 메소드 영역에서 선언할 경우에는 자동으로 값이 할당되지 않기 때문에 초기화를 해주어야 한다.
